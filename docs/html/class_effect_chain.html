Detailed Description

EffectChain owns an ordered sequence of audio effects and processes an AudioBuffer through them sequentially on each callback.

This class implements the Chain of Responsibility pattern - each effect in the chain receives the buffer in order and decides independently whether to process it or pass it through (based on its enabled state).

Thread safety:

  • All structural modification methods (add, remove, move) must be called from the GUI thread only, and only while the audio engine is stopped. Hot-swapping during a live session is not supported.
  • process() is called exclusively from the audio thread. No structural modifications may occur concurrently with process().
  • setEffectEnabled() is the only method safe to call from the GUI thread during an active session, as the enabled flag is atomic inside AudioEffectBase.
